# Refactor Workflow

Refactor code in small, verified steps.

## Steps

1. Identify target files and current behavior.
2. Define a minimal refactor plan.
3. Apply edits incrementally.
4. Run checks and tests.
5. Summarize changes and remaining risks.

## Exit Criteria

- Build and checks pass.
- Behavior remains unchanged unless explicitly requested.
- No scope creep outside requested refactor targets.
