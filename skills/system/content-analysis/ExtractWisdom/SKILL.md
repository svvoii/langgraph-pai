# ExtractWisdom

Dynamic content extraction flow for long-form source material.

## When To Use

- Extracting wisdom from a video, podcast, interview, or article
- Producing key takeaways and insight reports
- Summarizing source material while preserving high-signal insights

## Depth Levels

| Level | Sections | Bullets/Section | Closing Sections | Use Case |
|---|---|---|---|---|
| Instant | 1 | 8 | None | Quick scan |
| Fast | 3 | 3 | None | Short overview |
| Basic | 3 | 5 | One-Sentence Takeaway | Balanced summary |
| Full | 5-12 | 3-15 | All core closings | Default deep extraction |
| Comprehensive | 10-15 | 8-15 | All closings + Themes | Maximum depth |

## Voice Rules

- Write in direct spoken language, not report language.
- Prefer specific details over generic abstractions.
- Use short and medium-length sentences with varied rhythm.
- Avoid hedging phrases such as "the speaker noted".
- Preserve notable original phrasing when it is strong.

## Dynamic Section Rules

- Select section names from the actual content domains.
- Ensure each section has enough strong insights to justify inclusion.
- Include memorable quotes when available.
- Include first-time revelations when genuinely present.
- Avoid inventory-only sections that list facts without insight.

## Closing Sections

- One-Sentence Takeaway
- If You Only Have 2 Minutes
- References and Rabbit Holes
